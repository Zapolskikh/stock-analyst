**Проблема:** classifier пытается угадать сектор по финансовым метрикам, хотя Yahoo Finance уже предоставляет `nd.sector` и `nd.industry`. Из-за этого STZ (Constellation Brands) классифицируется как Mature Tech вместо Dividend/Defensive — у неё высокая gross margin "похожа" на tech compounder.

**Правильная архитектура (3 шага):**

### Шаг 1 — Yahoo sector → базовый CompanyType (авторитетно)
Заменить всё rule-based scoring для stable типов на прямой маппинг из `nd.sector`:
```
Yahoo "Technology" / "Communication Services"           →  MATURE_TECH
Yahoo "Healthcare"                                      →  PHARMA
Yahoo "Consumer Defensive" / "Utilities" / "Real Estate" →  DIVIDEND_DEFENSIVE
Yahoo "Energy" / "Materials" / "Industrials" / "Consumer Cyclical" →  CYCLICAL
Yahoo "Financial Services"                              →  FINANCIAL
```
Если `nd.sector` есть — это авторитетный ответ, скоринг не нужен.

### Шаг 2 — TURNAROUND override (финансовое состояние, не сектор)
После определения базового типа — проверить дистресс-сигналы:
- 2+ убыточных года из последних 3 → signal
- падение выручки с последующим восстановлением → signal
- D/E > 2.0 → signal

Если сигналов достаточно → переопределить на TURNAROUND **независимо от сектора**.
Это единственный случай, когда финансовые метрики перевешивают Yahoo.

### Шаг 3 — HYPERGROWTH_TECH override
Если базовый тип = MATURE_TECH И среднегодовой рост выручки > 25% за 3 года → переопределить на HYPERGROWTH_TECH.
Это единственное различие которое Yahoo не видит (оба типа = "Technology").
